Java核心技术之——对象方法的执行过程 + 动态绑定

Posted on 2020-01-28

动态绑定

一个对象变量在运行时能够自动地选择调用哪个类型方法的现象称为动态绑定。(dynamic binding)

实例

变量x声明为C类的对象，实际类型为D。

步骤1：

编译器查看对象的声明类型和方法名( 声明类型为C，方法名为f )
编译器列举所有 C 类中名为 f 和其超类中访问属性为 public 的方法（超类的私有方法不可访问）
至此，编译器已获得所以可能被调用的候选方法

步骤2：

重载解析：
编译器查看调用方法时提供的参数类型，如果在所有名为 f 的方法中存在一个与之提供的参数类型完全匹配，就选择这个方法。

如果编译器没有找到与参数类型匹配的方法，或者发现经过类型转换后有多个方法与之匹配，就会报告一个错误。

至此，编译器已获得需要调用的方法名字和参数类型。
覆盖(重写)

步骤3：

如果是 private 方法、static 方法、 final 方法或者构造器 ，那么编译器将可以准确地知道应该调用哪个方法，将这种调用方式称为静态绑定。

对应的，调用的方法依赖于 隐式参数 的实际类型,并且在运行时实现 动态绑定。

步骤4：

采用 动态绑定 时，虚拟机一定调用与 x 所引用对象的 实际类型（D） 最适合的那个类的方法。（D 有这个方法就直接用，没有在超类找）

简单总结

成员函数在多态调用时，编译看左边，运行看右边。

Java核心技术之——子类的实例化过程

Posted on 2020-01-28

过程

子类中的所有构造函数，默认都会访问父类中空参数的构造函数。因为子类每一个构造函数内的第一行都会有一句隐式的 super()。

当父类中没有空参数的构造函数时，子类必须通过手动 super 或者 this 语句来指定要访问的构造函数。（如果父类没有不带参数的构造器，在子类的构造器中又没有显式地调用父类的其他构造器，则Java编译器将报告错误）

为什么子类一定要访问父类中的构造函数？

因为父类中的数据子类可以直接获取(非私有)，所以子类对象建立时，需要先查看父类是如何对这些数据进行初始化的。所以子类在对象初始化时，要先访问一下父类中的构造函数。

例子

调用父类构造函数: super(…)
调用父类一般函数：super.f

特例

当然，子类的构造函数第一行也可以指定 this 语句访问本类中的构造函数。子类中至少会有一个构造函数会访问父类中的构造函数。

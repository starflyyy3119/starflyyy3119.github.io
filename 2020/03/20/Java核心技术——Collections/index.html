集合框架

  • 集合框架是一个用来代表和操纵集合的统一架构。
  • 整个集合框架围绕一组标准接口设计。这些接口存在标准实现。如：LinkedList, HashSet, 和 TreeSet 等。除此，可以通过这些接口实现自己的集合。
  • Java 集合框架主要包括两种类型的容器，一种是集合 (Collection), 储存一个元素集合; 另一种是图 (Map), 存储键/值对映射。
  • 所有的集合框架包含以下内容：
    — 接口： 是代表集合的抽象数据类型。如：Collection、List、Set、Queue、Map 等。定义多个接口，是为了以不同的方式操纵集合对象。
    — 实现(类)： 是集合接口的具体实现。它们可以重复使用的数据结构，如： ArrayList、LinkedList、HashSet、HashMap.
    — 算法： 是实现集合接口的对象里的方法执行的一些有用的计算，如：搜索和排序。

  • Java 集合框架位于 java.util 包中，使用集合框架时需要导入包。

Note: 数组和集合的区别

  • 数组长度不可变化而且无法保存具有映射关系的数据；集合类用于保存数量不确定的数据，以及保存具有映射关系的数据。
  • 数组元素既可以是基本类型的值，也可以是对象; 集合只能保存对象。

集合接口

接口 接口描述
Collection 最基本的集合接口，一个 Collection 代表一组 Object。 Java 不提供直接继承自 Collection 的类， 只提供继承于的子接口(如 List 和 Set)。Collection 接口储存一组不唯一、无序的对象。
List List接口是一个有序的 Collection，使用此接口能精确的控制每个元素插入的位置，能够通过索引（元素在 List 中位置，类似于数组的下标）来访问 List 中的元素，第一个元素的索引为 0，而且允许有相同的元素。List 接口存储一组不唯一，有序(插入顺序)的对象。
Set Set接口存储一组唯一，无序的对象。
Queue Queue接口是一种先进先出的数据结构，并提供插入、提取、检验等操作。
Map Map 接口存储一组键值对象，提供 key(键) 到 value(值) 的映射

Note: Set 和 List 的区别

  • Set 接口实例存储是无序的，不重复的数据。List 接口实例存储的是有序的，可以重复的元素。
  • Set 检索效率低下，删除和插入效率高，插入和删除不会引起元素位置改变，实现类有 HashSet， TreeSet。
  • List 和数组类似，但可以动态增长。根据实际存储的数据的长度自动增长 List 的长度。查找元素效率高，插入删除效率低，因为会引起其他元素的位置改变，实现类有 ArrayList，LinkedList， Vector。

集合实现类（集合类）

Java提供了一套实现了集合接口的标准集合类。其中一些是具体类，这些类可以直接使用，另外一些是抽象类，提供了接口的部分实现。
|集合实现类|特点|
|-|-|
|ArrayList|实现了List接口。使用数组存储元素，数组可以动态创建，如果元素个数超过了数组的容量，那么就增长当前长度的50%，并将当前数组中的所有元素都复制到新的数组中。该类非同步，在多线程的情况下不要使用。插入删除效率低。|
|LinkedList|实现了List接口。允许有 null 元素。主要用于创建链表数据结构。该类没有同步方法，如果多个线程同时访问一个 List，必须自己实现访问同步。|
|HashSet|实现了Set接口，不允许出现重复元素，不保证集合中元素的顺序，允许包含值为 null 的元素，但最多只能一个。|LinkedHashSet|继承了 HashSet，支持对规则集内的元素排序。HashSet 中的元素是没有被排序的，而 LinkedHashSet 中的元素可以按照它们插入规则集的顺序提取。|
|TreeSet|实现了Set接口，可以实现排序等功能。|
|HashMap|实现了Map接口。HashMap是一个散列表，它存储的内容是键值对(key-value) 映射。根据键的 HashCode 值存储数据，具有很快的访问速度，最多允许一条记录为null， 不支持线程同步。在 JDK1.8 中， HashMap 采用数组 + 链表 + 红黑树实现，当链表长度超过阈值(8) 时，将链表转化成红黑树。|
|LinkedHashMap|继承HashMap，主要用链表扩展 HashMap 类， HashMap 中条目没有顺序，而在 LinkedHashMap 中元素可以按照它们插入图的顺序排序，也可以按照它们最后一次被访问的顺序排序。|
|TreeMap|继承AbstractMap,基于红黑树实现。同时实现了接口 NavigableMap, 而NavigableMap 继承 SortedMap, 使用 SortedMap 确保排好序。键值可以使用 Comparable 或 Comparator 接口来排序|

Note: 在实际使用中，如果更新图时不需要保持图中元素的顺序，就使用 HashMap，如果需要保持图中元素插入顺序或者访问顺序，就使用 LinkedHashMap, 如果需要按照键值排序，就使用 TreeMap。

其他集合类

集合类 特性
Vector 同步，可以用于多线程，该类允许设置默认的增长长度，默认扩容方式为原来的2倍。
Stack Vector的子类，用来实现一个标准的后进先出的栈。
Dictionary 抽象类，用来储存键/值对。
Hashtable 继承Dictionary。函数都是同步的，线程安全。另外 Hashtable 中的 key 和 value 都不可以为 null。
Properites Hashtable 的子类，相当于一个 key， value 都是 String 类型的 Map，主要用于读取配置文件。
